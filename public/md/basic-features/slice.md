# 切片 (slice)

## 概述

切片是 Go 中最常用的数据类型之一，是对数组的抽象，提供了更灵活、更强大的序列操作能力。

## 切片的结构

```go
// 内部结构（运行时）
type slice struct {
    ptr unsafe.Pointer // 指向底层数组的指针
    len int            // 长度
    cap int            // 容量
}
```

## 创建切片

```go
// 1. 字面量
s := []int{1, 2, 3, 4, 5}

// 2. make 创建
s := make([]int, 5, 10) // 长度5，容量10

// 3. 从数组/切片切取
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:4] // [2, 3, 4]

// 4. nil 切片
var s []int // nil 切片，len=0, cap=0
```

## 切片操作

### 追加元素
```go
s := []int{1, 2, 3}
s = append(s, 4)        // [1 2 3 4]
s = append(s, 5, 6)     // [1 2 3 4 5 6]
```

### 切片拼接
```go
a := []int{1, 2}
b := []int{3, 4}
a = append(a, b...)     // [1 2 3 4]
```

### 删除元素
```go
// 删除索引 i 的元素
s = append(s[:i], s[i+1:]...)
```

### 复制切片
```go
dst := make([]int, len(src))
copy(dst, src)
```

## 扩容机制

当 `append` 超过容量时：

1. **新容量**：通常为旧容量的 2 倍左右（具体策略因 Go 版本而异）
2. **内存分配**：分配新的底层数组
3. **数据拷贝**：将旧数组内容复制到新数组

### 扩容策略 (Go 1.18+)

- 容量 < 256: 新容量 = 旧容量 * 2
- 容量 >= 256: 新容量 = 旧容量 + (旧容量 * 3 / 8) / 256 * 256

## 常见陷阱

### 1. 切片共享底层数组
```go
a := []int{1, 2, 3, 4, 5}
b := a[1:3] // b 和 a 共享同一个底层数组
b[0] = 99    // a 也变成了 [1, 99, 3, 4, 5]
```

### 2. append 导致的意外修改
```go
a := []int{1, 2, 3, 4, 5}
b := a[1:3]
b = append(b, 99) // 可能影响 a！如果 b 有足够容量
```

### 3. for range 的值拷贝
```go
s := []MyStruct{{}, {}, {}}
for _, v := range s {
    v.field = 1 // 不会影响原切片！
}
```
